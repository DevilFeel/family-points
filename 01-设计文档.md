# Family Points 离线版设计文档 (v1.0)

## 1. 项目概述
本工具是一款专为家庭设计的“积分激励”单机 Web 应用。通过 PWA 技术，实现在荣耀等安卓手机上的类原生 App 体验。数据完全存储于本地，无需联网，保护隐私且响应极快。

---

## 2. 核心架构
*   **前端框架**：Next.js (SSG 静态导出模式)。
*   **本地数据库**：IndexedDB (基于 **Dexie.js** 封装)。
*   **离线技术**：Service Workers + Manifest.json (PWA 标准)。
*   **UI 规范**：Tailwind CSS + Shadcn UI（适配移动端，支持深色模式）。

---

## 3. 页面与交互设计

### 3.1 家长管理页 (Parent Console)
*   **身份验证**：简易数字锁（防止孩子误操作）。
*   **任务卡片流**：
    *   预设常用任务（如：早起、阅读、收拾房间）。
    *   点击卡片立即加分，并伴有“成功”音效/视觉反馈。
*   **手动修正**：支持输入自定义分值和备注。
*   **审批流**：处理孩子发起的兑换请求。

### 3.2 孩子展示页 (Child Dashboard)
*   **核心视觉**：超大字号显示当前总积分。
*   **动态反馈**：积分增加时，数字滚动并伴有粒子/金币散落动画。
*   **等级/目标进度**：
    *   显示当前距离“终极奖励”的线性进度条。
    *   阶段性成就勋章展示。
*   **兑换按钮**：点击跳转至奖励商店。

### 3.3 数据管理中心 (Settings)
*   **导出功能**：一键生成 `points_backup_日期.json` 并调用系统分享发送给微信/网盘。
*   **导入功能**：支持读取备份文件以恢复数据。
*   **存储状态**：显示当前数据库占用空间及“永久存储”权限状态。

---

## 4. 数据库模型 (Schema)

| 表名 | 字段 | 说明 |
| :--- | :--- | :--- |
| **profiles** | `id`, `name`, `balance`, `target_points`, `target_name` | 存储孩子信息、当前余利及目标 |
| **tasks** | `id`, `title`, `points`, `icon`, `sort_order` | 存储家长预设的加分项目 |
| **logs** | `id`, `amount`, `reason`, `type`, `timestamp` | 积分变动的所有流水记录 |
| **rewards** | `id`, `title`, `cost`, `status(pending/done)` | 存储兑换奖励的记录 |

---

## 5. 安全与持久化策略

*   **数据持久化**：
    *   调用 `navigator.storage.persist()` 请求系统禁止自动清理数据。
    *   使用 Dexie 的 `Version` 管理，确保后续版本升级（增加新功能）时数据不丢失。
*   **离线保障**：
    *   配置 `next-pwa` 插件，确保应用在飞行模式下仍能通过桌面图标正常打开。
    *   本地缓存所有图标素材，避免因网络缺失导致 UI 出现“碎图”。

---

## 6. 针对荣耀 (MagicOS) 的优化建议

1.  **添加到桌面**：建议使用荣耀自带浏览器或 Chrome。
2.  **防止清理**：在多任务管理界面，可以将“Family Points”卡片下拉加锁，防止系统激进清理内存。
3.  **权限**：如果需要导出文件，确保浏览器拥有“存储/文件访问”权限。

## 7. 开发路线图 (Roadmap)

1.  **Phase 1 (MVP)**：完成本地数据库搭建，实现简单的加分/减分和总分显示。

3.  **Phase 3 (Backup)**：完成 JSON 导出与恢复功能，解决数据迁移后顾之忧。
4.  **Phase 4 (Future)**：(可选) 增加 Supabase 插件，支持多端数据实时同步。

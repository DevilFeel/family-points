# Family Points 现状设计说明

## 1. 项目定位

`Family Points` 是一个离线优先的家庭积分应用，主要用于家长给孩子记录日常表现、奖励和兑换情况。当前版本以“够用、稳定、上手快”为目标，不引入复杂账号体系或联网能力。

## 2. 当前能力范围

### 首页

- 展示孩子当前积分
- 点击任务卡快速加分
- 支持自定义加分和扣分
- 支持奖励兑换
- 展示今日积分记录

### 家长管理页

- 查看当前积分
- 新增、编辑、删除任务
- 新增、删除奖励
- 执行奖励兑换
- 查看累计、今日、近 7 天统计
- 查看和删除操作记录

### 设置页

- 修改孩子名称
- 导出 JSON 备份
- 导入 JSON 备份
- 清空所有本地数据

## 3. 技术方案

- 前端框架：Next.js 14
- 本地存储：IndexedDB + Dexie
- 界面样式：Tailwind CSS + 基础 UI 组件
- 动画：Framer Motion
- 离线能力：PWA + Service Worker

## 4. 数据模型

### profiles

- `id`
- `name`
- `balance`

### tasks

- `id`
- `title`
- `points`
- `icon`
- `sortOrder`

### logs

- `id`
- `profileId`
- `amount`
- `reason`
- `type`
- `timestamp`

### rewards

- `id`
- `title`
- `cost`
- `icon`
- `enabled`

## 5. 当前约束

- 当前按单个孩子档案设计
- 数据完全保存在本地
- 不包含多设备同步
- 不包含积分目标字段
- 不包含权限系统和后端服务

## 6. 优化方向

当前更适合围绕以下方向持续优化：

- 保持文档和代码一致
- 保持页面组件边界清晰
- 保持文案和默认数据完整可读
- 在不增加复杂度的前提下改进移动端体验
